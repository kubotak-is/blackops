# Inline Dispatcher

Public DispatcherはOperation DefinitionとOperationValueを受け取り、OperationResultを返す。PSR-11 ContainerやInternal Factoryは公開Signatureへ露出しない。

Internal InlineDispatcherはRegistryからMetadataを解決し、受信Contextと最初のAttemptを生成してOperationEnvelopeを構築する。HandlerResolverだけがPSR-11 Containerへアクセスする。

Inline実行はHandlerを一度だけ呼び、自動Retryしない。Keyless Handler例外はResultへ変換せず、後続のLifecycle／Supervision境界へ伝播する。Keyed PHP direct実行ではFailure Boundaryが安全なInternal Failure Recordを保存して`OperationExecutionFailed`を投げる。HTTP Error Boundaryだけがその失敗を安全な500 Response Snapshotへ変換・Attachし、同じHTTP KeyのDuplicateでReplayする。

Keyed MutationはAuthorization後にScope／Fingerprintを計算してAtomic Claimする。同FingerprintのProcessingはIn-progress、異なるFingerprintはConflict、TerminalはTyped ResultまたはSafe HTTP SnapshotのReplayとなる。必要なSnapshotやResultが失われたTerminal RecordはExpiredとして再実行しない。

InlineDispatcherはExecution Scope内でLifecycleStateを保持し、Journal RecordをWriterへ渡す前にState Machineで遷移を検証する。正常完了ではReceived、Running、Finalizing、Completedへ進み、業務拒否ではRunningからRejectedへ進む。

Ephemeral Metadataでは、Binding済みValueをReceived Recordへ渡さず`EmptyJournalData`へ置換する。Handlerの実`EphemeralOutcome`は呼出元へ返すが、Completed Recordには`OperationCompletedData(new EmptyOutcome())`だけを渡す。Key付きEphemeral RouteはClaim前に拒否する。ShapeとJSON Encoding可能性の検証はHandler実行後かつTransaction Commit前に行い、Canonical Writer、Observer、Outcome Storeへ実Objectを到達させない。
