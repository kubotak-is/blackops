# Journal Ports

CanonicalJournalWriterとCanonicalJournalReaderは書き込みと読み取りの責務を分離する。両方を提供するAdapterはCanonicalJournalStoreを実装する。

Writerは成功時にvoidを返し、失敗時はJournalWriteFailedを投げる。InlineSequenceは一つのExecution Scope内で1から始まる単調増加Sequenceを割り当てる。

## Observer Ports

ObserverはCanonical JournalRecordではなくObservedJournalRecordだけを受け取る。ObservedJournalRecordは同じ論理Envelopeを保持するが、dataはprojection済みarrayであり、Raw JournalData objectは保持しない。

JournalObserverは成功時にvoidを返し、失敗時はJournalObservationFailedを投げる。Buffer型ObserverはFlushableJournalObserverを追加で実装し、Operation終了、Worker Loop終了、Process Shutdown等の境界でflushされる。

Canonical JournalRecordからObservedJournalRecordへの変換はInternal projectorが行い、Sensitive Projection Filterを通してからObserverへ渡す。

Internal Observer Aggregatorは複数Observerを独立して実行し、個別の成功と失敗を集約する。BestEffortはObserver失敗をOperation継続可能な失敗として記録し、RequiredとDurableは全Observerの試行後にJournalObservationFailedとして扱う。
